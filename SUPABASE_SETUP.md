# Supabase Setup Guide\n\nThis document walks through setting up a Supabase backend for the Hidroponik project.\n\n## 1. Create a Supabase Project\n\n1. Go to [supabase.com](https://supabase.com) and sign up (free tier available)\n2. Click **\"New Project\"**\n3. Choose a name, set a strong password, select your region\n4. Wait for the project to initialize (~2 min)\n\n## 2. Create the `devices` Table\n\nIn the Supabase dashboard:\n\n1. Click **\"SQL Editor\"** (left sidebar)\n2. Click **\"New Query\"**\n3. Paste this SQL:\n\n```sql\nCREATE TABLE devices (\n  device_id TEXT PRIMARY KEY,\n  led_start TEXT DEFAULT '08:00',\n  led_end TEXT DEFAULT '20:00',\n  led_override BOOLEAN,\n  pump_slots TEXT DEFAULT '08:00,20:00',\n  pump_override BOOLEAN,\n  temperature FLOAT,\n  humidity FLOAT,\n  water_level INTEGER,\n  tds FLOAT,\n  clock_time TIMESTAMP,\n  last_seen TIMESTAMP DEFAULT NOW(),\n  power_status BOOLEAN,\n  wifi_status BOOLEAN,\n  pump_status BOOLEAN,\n  led_status BOOLEAN,\n  created_at TIMESTAMP DEFAULT NOW(),\n  updated_at TIMESTAMP DEFAULT NOW()\n);\n\nALTER TABLE devices ENABLE ROW LEVEL SECURITY;\n\nCREATE POLICY \"Enable read access for all\" ON \"public\".\"devices\"\nAS PERMISSIVE FOR SELECT USING (true);\n\nCREATE POLICY \"Enable update access for all\" ON \"public\".\"devices\"\nAS PERMISSIVE FOR UPDATE USING (true);\n\nCREATE POLICY \"Enable insert access for all\" ON \"public\".\"devices\"\nAS PERMISSIVE FOR INSERT WITH CHECK (true);\n```\n\n4. Click **\"Run\"**\n\n## 3. Get Your Credentials\n\n1. Click **\"Settings\"** (bottom left) â†’ **\"API\"**\n2. Copy:\n   - **Project URL** â†’ paste into `SUPABASE_URL` in `config.h`\n   - **anon (public)** key â†’ paste into `SUPABASE_KEY` in `config.h`\n3. Save `config.h`\n\n## 4. Test the Connection\n\n1. Upload firmware to ESP32\n2. Open **Serial Monitor** (115200 baud)\n3. On first boot, device will:\n   - Scan WiFi networks\n   - Show a captive portal (`Hidroponik-<MAC>`)\n   - Register itself in Supabase\n\n4. Check the `devices` table in Supabase:\n   - Should see a new row with your device ID\n   - Monitor `last_seen` â€” should update every ~10 seconds\n\n## 5. Control Your Device\n\nEdit the row for your device in Supabase:\n\n| Column | Value | Effect |\n|--------|-------|--------|\n| `led_start` | `\"08:00\"` | LED turns on at 8 AM |\n| `led_end` | `\"20:00\"` | LED turns off at 8 PM |\n| `pump_slots` | `\"06:00,12:00,18:00\"` | Pump runs at these times (5-min windows) |\n| `led_override` | `null` / `true` / `false` | Manual override: null=schedule, true=force on, false=force off |\n| `pump_override` | `null` / `true` / `false` | Same for pump |\n\n\n## 6. View Telemetry\n\nDevice sends sensor data every 10 seconds:\n- `temperature`, `humidity`, `water_level`, `tds`\n- `power_status`, `wifi_status`, `pump_status`, `led_status`\n- `clock_time`, `last_seen`\n\nYou can query or visualize this in Supabase.\n\n---\n\n## Troubleshooting\n\n**Device won't connect to Supabase:**\n- Check URL/key in `config.h`\n- Verify WiFi is connected (Serial shows `[WIFI] OK`)\n- Ensure table `devices` exists in your Supabase project\n- Check Row Level Security policies are permissive (as shown above)\n\n**Serial shows `[DB] Cihaz kontrol: 401`:**\n- Invalid `SUPABASE_KEY` â€” regenerate from Settings > API\n\n**Device registers but data doesn't appear:**\n- Wait ~10 sec, refresh Supabase dashboard\n- Check `last_seen` timestamp updates\n- If not, inspect Serial for HTTP errors\n\n---\n\nOnce set up, your device will automatically:\n- Report sensor readings every 10 seconds\n- Fetch schedule updates every 5 seconds\n- Respond to manual overrides in real-time\n"
