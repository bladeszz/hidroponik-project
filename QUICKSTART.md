# Quick Start — 5 Minutes\n\nGet your hydroponic system running in 5 minutes.\n\n## Prerequisites\n\n✓ Arduino IDE or PlatformIO\n✓ ESP32 board support installed\n✓ USB cable for ESP32\n\n## Step 1: Install Libraries\n\nIn Arduino IDE, go to **Sketch > Include Library > Manage Libraries** and install:\n- `WiFiManager` by Tzapu\n- `ArduinoJson` by Benoit Blanchon\n- `RtcDS1302` by Makuna\n- `DHT sensor library` by Adafruit\n\n## Step 2: Configure Credentials\n\n```bash\ncp config.h.example config.h\n```\n\nEdit `config.h` and fill in:\n```cpp\nconst char* SUPABASE_URL = \"https://YOUR_PROJECT.supabase.co\";\nconst char* SUPABASE_KEY = \"YOUR_ANON_KEY\";\n```\n\n**Don't have Supabase yet?**\n→ See [SUPABASE_SETUP.md](SUPABASE_SETUP.md)\n\n## Step 3: Upload\n\n1. Plug ESP32 into USB\n2. Select **Tools > Board: ESP32 Dev Module**\n3. Click **Upload**\n4. Open **Serial Monitor** (115200 baud) to see logs\n\n## Step 4: First Boot — WiFi Setup\n\nOn first boot, the device:\n1. Creates a WiFi AP called `Hidroponik-<deviceID>`\n2. Waits for you to connect\n3. Shows a captive portal for WiFi config\n\n**To connect:**\n- Connect to `Hidroponik-XXXXXX` WiFi (password: `topraksiz`)\n- A portal should auto-open; select your WiFi and enter password\n- Device registers itself in Supabase\n\n## Step 5: Monitor & Control\n\n**See live data:**\n- Open Serial Monitor → sensor readings every 2 seconds\n- Open Supabase dashboard → `devices` table → your device row\n\n**Control remotely:**\n- Edit `led_start`, `led_end`, `pump_slots` in Supabase\n- Changes apply within ~5 seconds\n\n---\n\n## Manual WiFi Reset\n\nIf WiFi stops working or you want to reconfigure:\n\n1. Hold **BOOT button on ESP32 for 3 seconds**\n2. Device reboots and clears WiFi settings\n3. Captive portal appears again\n\n---\n\n## Next Steps\n\n- [Full README](README.md) — detailed docs, troubleshooting\n- [Supabase Setup Guide](SUPABASE_SETUP.md) — database schema & SQL\n- [config.h](config.h.example) — pin definitions, timing tweaks\n\n---\n\n**Questions?** Check the [README](README.md#troubleshooting) troubleshooting section.\n"
